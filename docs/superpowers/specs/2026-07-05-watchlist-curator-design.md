# Watchlist Curator — Design

**Date:** 2026-07-05
**Status:** Approved

## Purpose

A daily + weekly automation that (a) manages YouTube playlists of the best 2-3 videos per day (≤1 hr total watch time) from Chris's curated subscriptions, and (b) publishes a newsletter-style writeup — headlines, breaking news, video dissections, GitHub repo watchlist — covering local LLMs, Snowflake, dbt, and LLM development/news.

## Decisions (settled with user)

- **Runtime:** GitHub Actions cron workflows (daily + weekly). Runs independent of Chris's PC.
- **Delivery:** Markdown newsletters committed to the repo, rendered via GitHub Pages (Jekyll default, zero build config).
- **Video source:** Chris's YouTube subscriptions, filtered to a curated channel list generated once by Claude and hand-editable thereafter.
- **News sources:** Curated RSS feeds + Perplexity API research call at runtime + GitHub trending.
- **Intelligence:** Plain Python does all fetching; 1-2 Claude API calls per run do ranking and newsletter writing. No agentic runtime.
- **Credentials:** Anthropic API key exists. Google Cloud / YouTube Data API needs one-time setup (guide included). Perplexity API key required.

## Architecture

Single repo, single Python package, two entrypoints. No database — the repo is the state.

```
config/interests.yaml      # topics, weights, phrasing passed to the LLM
config/channels.yaml       # curated channel IDs (generated by setup script, hand-editable)
config/feeds.yaml          # RSS sources (HN, r/LocalLLaMA, dbt/Snowflake/Anthropic blogs, ...)
src/youtube.py             # subscriptions, video metadata, auto-captions, playlist CRUD
src/sources.py             # RSS pull, Perplexity research call, GitHub trending (topic-filtered)
src/curate.py              # Claude API calls: rank videos, write newsletter (structured JSON out)
src/daily.py               # daily entrypoint (supports --dry-run)
src/weekly.py              # weekly entrypoint (supports --dry-run)
scripts/setup_auth.py      # one-time Google OAuth consent flow → refresh token
scripts/curate_channels.py # one-time: pull all subscriptions → Claude classifies → channels.yaml
site/                      # newsletters as dated markdown, served by GitHub Pages
.github/workflows/daily.yml
.github/workflows/weekly.yml
```

## Daily run (cron, ~6am Europe/London)

1. Fetch last-24h uploads from channels in `channels.yaml` (title, description, duration).
2. Fetch RSS items from `feeds.yaml` (last 24h).
3. One Perplexity query: notable news today in local LLMs / Snowflake / dbt / LLM development.
4. Fetch GitHub trending repos filtered by topic keywords.
5. Pull YouTube auto-captions for candidate videos (no audio transcription; videos without captions get title+description treatment only).
6. One Claude call with all material + `interests.yaml` + dedupe list of previously recommended video IDs (grepped from past newsletters). Returns structured JSON:
   - `playlist_videos`: top 2-3, total ≤60 min
   - `dissect_videos`: lower-priority videos summarized from captions
   - `newsletter`: sections — Breaking News (omitted unless warranted), Headlines, Video Dissections, Repo Watchlist ("why it's cool"), all with citation links. Target read: 20-30 min max for weekly, shorter daily.
7. Add `playlist_videos` to the current week's playlist.
8. Delete playlists older than 30 days (age parsed from playlist name).
9. Commit `site/newsletters/YYYY-MM-DD.md`; Pages publishes.

## Weekly run (Sunday, before daily)

1. Create the week's playlist, named as the Sunday start date `DD-MM-YYYY` (e.g. `05-07-2026`).
2. Write a weekly digest edition synthesizing the past week's daily newsletters (read from `site/newsletters/`).

## Breaking news

The Claude prompt decides: if source material contains a major release, launch, or shutdown (e.g. a model launch/retirement), it writes a Breaking News section including practical "how to make the most of it" examples. Otherwise the section is omitted entirely.

## Error handling

- YouTube API failure → newsletter still publishes, playlist step skipped (and vice versa).
- Actions job failure → GitHub's built-in failure notification email. No retry logic.

## Secrets (GitHub Actions secrets)

`ANTHROPIC_API_KEY`, `PERPLEXITY_API_KEY`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`.

## State

None beyond the repo. Playlist retention derives from playlist names; dedupe derives from past newsletter files.

## Testing

- One small self-check per `src/` module (assert-based, no framework).
- `python -m src.daily --dry-run` runs the full pipeline locally, prints the newsletter, mutates nothing (no playlist changes, no commits).

## Explicitly out of scope

Database, web UI, email delivery, audio transcription, retries/queues, multi-user support.
